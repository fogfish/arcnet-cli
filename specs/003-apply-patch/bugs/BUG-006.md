# Bug Report: BUG-005's "refuse the whole patch" fix rejects a real, valid patch that legitimately carries a `kind: timeline` section

**Type**: Implementation drift (of a same-day spec correction)
**Severity**: Critical
**Feature**: 003-apply-patch (`arc apply`)
**Reported**: 2026-07-04
**Status**: Resolved
**Patched**: 2026-07-04
**Resolved**: 2026-07-04 — see tasks.md T102-T106

## Description

Minutes after [BUG-005](BUG-005.md) was marked resolved, applying a real,
previously-working patch regressed:

```text
$ arc apply email-dev-thought.md
❌ Reading patch file failed: [github.com/fogfish/arcnet-cli/internal/core.parsePatchBody 271] patch body does not follow the H1-kind/H2-node section structure
```

The patch (`github.com/fogfish/bots/email-dev-thought.md`) is a real,
well-formed document contribution: a `# Source` section for
`dmitry-2026-graph-thought`, several `# Entity` sections, `# Thought`
sections — and, at line 280, an explicit `# Timeline` section:

```markdown
# Timeline

## 2026-07

```yaml
granularity: monthly
```

- [[dmitry-2026-graph-thought]]
```

BUG-005's fix made `internal/core.ParsePatch`'s `parsePatchBody` reject
**any** patch whose body contains a `kind: timeline` H1 section, on the
assumption (drawn from `research.md` D8/spec.md FR-022's own aside,
"the timeline index is never itself patch-carried") that a patch would
never legitimately declare one. That assumption was wrong: the real
extraction pipeline that produces these patches intentionally emits a
`# Timeline` section as a self-describing period annotation alongside the
document's own `# Source` section. Refusing the entire patch broke real,
valid usage — a correctness regression more severe than the original
collision bug it was meant to fix, since it now blocks ingestion outright
rather than merely risking a stray duplicate file.

## Root Cause Analysis

BUG-005's own investigation correctly diagnosed the *mechanism* of the
original collision (an unmapped `"timeline"` kind falling through
`nodeFolder`'s pluralization fallback into a bogus `"timelines/"` folder)
but never validated its chosen *fix* against a real, in-the-wild patch —
it reasoned from this codebase's own prior documentation of intent
("index kinds are not carried") without checking whether that intent was
actually upheld by real patch producers. It was not: a real, external
extraction pipeline (outside this repository, in `github.com/fogfish/bots`)
treats an explicit `# Timeline` section as a legitimate, useful annotation.
The fix that would have avoided this regression — reported at all — is
the same discipline this project's constitution already asks for
elsewhere (Principle VIII, E2E acceptance testing against real scenarios):
BUG-005's tasks.md added only synthetic refusal tests (T098/T099), never
an E2E test applying a patch that resembles this real-world shape (Source
+ Timeline together), which would have caught the regression before it
shipped.

There is also a second, independently real defect BUG-005's original fix
would have re-introduced had a *later* attempt "corrected" it by simply
routing `kind: timeline` to the right folder instead of refusing it:
`applyTimeline`'s period files use a specialized front-matter-plus-
chronologically-sorted-bullet-list rendering (`upsertTimelinePeriod`/
`core.TimelineEntry`), never `core.RenderNode`'s generic node
serialization — a generically-rendered `timeline/<id>.md` file would be
unparseable by `applyTimeline`'s own `parseTimelineEntries` regex on the
next `arc apply` run touching that period. This was caught during this
bugfix's own design (research.md D8b's revised Rationale), not by a
separate report, but is recorded here since it directly shaped why the
correct fix folds the declared period into `applyTimeline`'s existing
mechanism rather than "just writing it to the right path."

## Recommended Fix

1. Revert `internal/core.ParsePatch`'s reserved-kind guard (T097).
2. In `internal/app/graph/service.Apply`, intercept `Kind == "timeline"`
   before the generic per-node create/merge path; fold its declared
   period into `applyTimeline`'s existing yearly/monthly derivation,
   extended to accept additional patch-declared periods with
   monthly→yearly cascade.
3. Replace the refusal-oriented unit/E2E tests with tests asserting
   successful application and correct index folding, including the
   cascade case.
4. Verify against the real, previously-failing patch
   (`github.com/fogfish/bots/email-dev-thought.md`) directly, not only
   via synthetic fixtures.
