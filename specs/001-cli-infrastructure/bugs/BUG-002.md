# Bug Report: GitHub Actions warn that Node 20 is deprecated across all three workflows

**Type**: Dependency issue
**Severity**: Low
**Feature**: 001-cli-infrastructure (branch `001-cli-infrastructure`)
**Reported**: 2026-07-01
**Status**: Patched
**Patched**: 2026-07-01

## Description

CI runs of all three workflows (`check-test.yml`, `check-code.yml`, `build.yml`) emit a GitHub Actions runner warning:

```
Node 20 is being deprecated. This workflow is running with Node 24 by default.
```

This warning is emitted by the Actions runner whenever a used action's `action.yml` still declares `runs.using: node20` — GitHub is deprecating the Node 20 runtime for actions in favor of Node 24. No job currently fails because of this; it is a forward-looking deprecation notice, not a runtime error. However, several actions pinned in this feature's workflows were pinned by major-version tag or dated release, without checking whether that pinned version has migrated to the Node 24 runtime:

- `actions/checkout@v4`
- `actions/setup-go@v5`
- `shogo82148/actions-goveralls@v1`
- `dominikh/staticcheck-action@v1.3.1`
- `reecetech/version-increment@2023.10.2` (a dated 2023 release, the most likely candidate to still be on Node 20)
- `goreleaser/goreleaser-action@v5`

None of these were checked against Node runtime support when pinned during implementation — the pinning decisions (research.md's "Decision: Coverage reporting", "Decision: Static analysis gate", "Decision: Versioning and release automation", "Decision: Vulnerability scanning" / GoReleaser shape) only recorded *which* action and *why*, never which runtime the pinned version uses.

## Artifact Traceability

### spec.md
- **Affected user story**: User Story 2 — Automated verification on every pull request (Priority: P2); User Story 3 — Automated versioned release on merge (Priority: P3) — all three workflows are affected
- **Affected requirements**:
  - FR-011: "All automated pipelines MUST pin the language toolchain version explicitly rather than resolving to an unpinned 'latest' version at run time" — this requirement pins the *Go* toolchain only; it has no equivalent for the Node runtime that third-party Actions execute under, which is exactly the gap this bug falls into
- **Gap identified**: spec.md has no requirement that CI action dependencies be kept current with the platform's supported Actions runtime (Node version). This is a genuine spec gap, not just an implementation slip — nothing in the spec would have prompted checking Node-runtime currency when T017/T018/T022 pinned these action versions.

### plan.md
- **Affected sections**: Project Structure (`.github/workflows/*`); no Technical Context line addresses third-party Action runtime currency
- **Impact**: No architectural change needed; this is a maintenance/currency concern for the CI workflow files already scoped by the plan, not a new component.

### research.md
- **Affected sections**: "Decision: Coverage reporting" (`shogo82148/actions-goveralls@v1`), "Decision: Static analysis gate" (`dominikh/staticcheck-action@v1.3.1`), "Decision: Versioning and release automation" (`reecetech/version-increment@2023.10.2`), "Decision: GoReleaser configuration shape" (`goreleaser/goreleaser-action@v5`) — each names an action version but none records a check against Node-runtime deprecation status at the time of pinning.

### tasks.md
- **Affected tasks**:
  - T017, T018 — created `check-test.yml`/`check-code.yml` pinning the actions above without a Node-runtime check
  - T022, T024 — created `build.yml` pinning `reecetech/version-increment@2023.10.2` and `goreleaser/goreleaser-action@v5` without a Node-runtime check
  - T019 — `actionlint` validation ran but `actionlint` does not detect Node-runtime deprecation warnings, so it could not have caught this
- **False completions**: None — T017/T018/T019/T022/T024 did exactly what they specified; the gap is in what they specified, not in whether they were executed correctly
- **Missing tasks**: A dependency-investigation task to check each pinned action's current major version for Node 24 runtime support, and bump any that are still on Node 20 where a newer major/tag is available

## Root Cause Analysis

This is a dependency issue rather than an implementation error: every action was pinned to a version documented as correct in research.md, and all workflows pass `actionlint` and run successfully today — the warning does not indicate incorrect configuration, only that some pinned action versions (most plausibly `reecetech/version-increment@2023.10.2`, given its 2023 vintage) have not yet been republished under the Node 24 runtime, or that we pinned an older major than one that has migrated. Because GitHub Actions' underlying runner Node version is decided by GitHub and by each action maintainer's own release cadence — not by anything in this repository — there was no way to catch this at implementation time other than an explicit, ongoing check, which no task currently owns.

## Recommended Fix

1. Run `/speckit.bugfix.patch` to add a requirement (or an explicit assumption) that CI action dependencies must track current supported Actions runtimes, and add a dependency-investigation task to `tasks.md` for checking/bumping the actions listed above
2. Run `/speckit.bugfix.verify` to confirm consistency
3. Resume `/speckit.implement` to apply the fix: for each pinned action, check whether a newer major/tag exists that has migrated to the Node 24 runtime and bump the pin if so; where no such version exists yet (upstream hasn't migrated), leave the pin as-is and note the warning as an accepted, upstream-owned limitation rather than something this repository can currently resolve
